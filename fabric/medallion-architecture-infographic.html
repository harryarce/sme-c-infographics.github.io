Medallion
Lakehouse Architecture

A progressive data refinement pattern that transforms raw data into trusted, analytics-ready insights — implemented natively in Microsoft Fabric with OneLake.

The Concept

What is Medallion Architecture?

A three-stage cleaning and organizing process for your data. Each layer makes data more reliable, structured, and ready for business use.

▵

Incremental Refinement

Data progresses from raw to enriched to curated. Each layer adds structure and quality, maintaining a clear lineage from source to insight.

⚖

ACID Compliance

Built on Delta Lake, every transaction guarantees Atomicity, Consistency, Isolation, and Durability — preventing data corruption at scale.

❖

Separation of Concerns

Each layer lives in its own lakehouse or workspace. Data engineers own Bronze/Silver; analysts and BI teams own Gold.

The Three Layers

Bronze → Silver → Gold

Click each layer to explore its purpose, tools, and best practices in Microsoft Fabric.

B

Bronze Layer

Raw Data · Landing Zone

Store everything exactly as it arrives. No changes allowed — preserve the source of truth.

What Goes Here

  • Raw JSON from web APIs
  • CSV exports from warehouses
  • SQL database snapshots
  • IoT telemetry streams
  • Unstructured files (PDFs, images)

Fabric Tools

  • Data Pipelines (Copy Activity)
  • Dataflow Gen2
  • OneLake Shortcuts (ADLS, S3, GCS)
  • Eventstream (real-time)

Storage Format

  • Original format preserved
  • Delta tables for relational sources
  • Parquet for columnar data
  • Files area for unstructured data

Best Practices

  • Never delete bronze data
  • Partition by ingestion date
  • Add audit columns (_ingestion_ts)
  • Use shortcuts instead of copying
  • Handle schema evolution
Cleanse · Validate · Deduplicate
S

Silver Layer

Enriched Data · Conformed Zone

Fix errors, standardize formats, remove duplicates, and match records across systems.

Transformations

  • Standardize date/time formats
  • Convert currencies (e.g., to USD)
  • Remove test/invalid transactions
  • Flatten nested JSON structures
  • Apply surrogate keys

Fabric Tools

  • Spark Notebooks (PySpark/SQL)
  • Dataflow Gen2
  • SQL Scripts
  • Materialized Lake Views

Storage Format

  • Delta tables (required)
  • V-Order optimization enabled
  • Liquid Clustering preferred
  • OPTIMIZE & VACUUM maintenance

Best Practices

  • Apply data quality rules here
  • Resolve heavy logic before Gold
  • Use MERGE for upserts
  • Target ~1 GB file sizes
  • Enforce conformed dimensions
Aggregate · Model · Optimize
G

Gold Layer

Curated Data · Business-Ready

Organize for reports and dashboards — star schemas, KPI tables, and department-specific data marts.

What Gets Built

  • Star/snowflake schema models
  • Fact & dimension tables
  • Aggregated KPI tables
  • Department data marts
  • Customer lifetime value tables

Fabric Tools

  • Data Warehouse (SQL endpoint)
  • Power BI Semantic Models
  • SQL Analytics Endpoint
  • Materialized Lake Views

Storage Format

  • Delta tables (optimized)
  • Warehouse tables for SQL
  • V-Order for fast reads
  • Large file sizes (~1 GB)

Best Practices

  • Clearly define table grain
  • Manage SCDs (slowly changing dims)
  • Pre-aggregate where appropriate
  • Enable self-service analytics
  • Consistent KPI definitions

Governance

Security at Every Layer

Built-in security capabilities in Fabric protect data from ingestion through consumption.

● Workspace Isolation

Each medallion layer in its own workspace for granular access control and governance.

● Role-Based Access

RBAC at the workspace, lakehouse, and warehouse level to restrict who sees what.

● Row-Level Security

Filter data dynamically by user role — each user sees only what they should.

● Object-Level Security

Restrict access to sensitive columns (e.g., PII, salary data) from unauthorized eyes.

● Data Lineage

Track every transformation from source to Bronze to Silver to Gold with full visibility.

● Sensitivity Labels

Classify confidential or regulated data with Microsoft Purview integration.

● Delta Time Travel

Roll back to any previous state for audit, compliance, and debugging purposes.

● Domains (Data Mesh)

Organize medallion layers within business domains for decentralized governance.

Optimization

Best Practices

Key recommendations from the official Microsoft guidance for a production-ready medallion implementation.

01
📦

Optimize File Sizes

Target ~1 GB files in Silver/Gold layers. Smaller files are fine in Bronze. Use OPTIMIZE command to compact small files.

02
📚

Use Liquid Clustering

Prefer Liquid Clustering over partitioning in Silver/Gold layers for better query performance. Partitioning works well in Bronze for ingestion patterns.

03
🔗

Shortcuts Over Copies

When source data is in ADLS Gen2, S3, or Google Cloud, create shortcuts in the Bronze layer instead of copying data across.

04
🕐

Manage History Retention

Use VACUUM to clean up old Delta history. Default 7-day retention prevents accidental data loss while controlling storage costs.

05
🏠

Separate Workspaces

Create each lakehouse in its own workspace for better control, governance, and access management at the layer level.

06
⚡

V-Order Everything

Fabric applies V-Order write-time optimization to Parquet automatically. This enables fast reads from Power BI, SQL, and Spark.

Advanced Pattern

Scale with Data Mesh

For massive, decentralized organizations, implement medallion architecture within Fabric Domains — mapping to your business areas (Marketing, Sales, HR). Each domain gets its own Bronze → Silver → Gold pipeline with federated governance.
