When single-surface governance matters

  • Regulated industries. Fewer control planes usually means fewer logs and policies to reconcile.
  • Incident response. GitHub-hosted agents keep issue, PR, and review state close to the code.
  • Access revocation. GitHub policy controls are simpler to audit than multiple disconnected vendor setups.

Where this story gets complicated

  • Content exclusion is the key caveat. Copilot cloud agent does not honor Copilot content exclusions, and GitHub-hosted third-party agents share those limitations. The Copilot desktop app inherits the same exclusion gap.
  • Claude Code Action can be attractive for runner control. The tradeoff is workflow and secret management per repo.
  • “Mixed” is not automatically bad. Vendor-side usage views may be valuable; they just add another evidence surface.
  • Agent 365 is the cross-cutting governance plane Microsoft is building. Announced at Build 2026, it unifies Entra identity, Defender threat signals, and Purview data-handling controls over agent activity — worth tracking even if it doesn’t replace these per-product surfaces today.
  • Agent governance changes quickly. Re-check this table before reusing it after major product updates.

About this infographic

An InfoSec / platform-governance view of AI coding agents around GitHub. The claim is narrow: GitHub-hosted agents concentrate the review workflow in GitHub; standalone vendor paths add useful controls but also additional evidence surfaces. Each pattern has tradeoffs.

Produced by a Microsoft employee. Views are my own; not an official Microsoft comparison and not endorsed by GitHub, Anthropic, OpenAI, Cursor, or Anysphere.
