5Apps & Surfaces

Where humans meet AI

The top of the stack — every app a person actually opens, from M365 to LOB to bespoke.

Microsoft 365 Copilot is embedded directly in Word, Excel, Outlook, Teams, and PowerPoint — but the surface layer is bigger than M365. Dynamics 365 brings agents into Sales, Service, and Finance. Power Platform (Power Apps, Power Automate, Power Pages) lets makers ship low-code apps that consume the same agents and models. Custom apps hosted on Azure App Service, Container Apps, or AKS round out the picture for bespoke experiences.

  • Outlook · Teams · Word · Excel · PowerPoint
  • Dynamics 365 · Power Apps · Power Automate · Power Pages
  • Custom apps on App Service / Container Apps / AKS

Edge to Edge

Two rails that touch every layer.

Identity and governance are not a step in the stack — they run alongside every layer, from the human surface down to the infrastructure. If you only ever remember two things on this page, remember these.

Identity

Every call, every agent, every model — needs an owner.
  • Microsoft Entra ID — user, group, conditional access for humans across M365 and Azure.
  • Entra Agent ID — non-human identity for agents, so every tool call is attributable.
  • Managed identities — backend-to-backend auth without secrets.
  • OAuth 2.1 native MCP auth in APIM (aligned with MCP spec 2025-06-18).
  • Credential Manager in APIM · outbound token injection with Key Vault references.
Docs: Microsoft Entra ID · Entra Agent ID

Governance & Trust

Policy you can write down, enforce, and audit.
  • Microsoft Purview — sensitivity labels, DLP, lineage, data lifecycle.
  • Microsoft Defender for Cloud — threat detection across the AI stack.
  • Azure AI Content Safety — prompt/completion moderation, AI-generated detection.
  • Responsible AI — evaluations, red-teaming, observability in Foundry.
  • Tiered agent governance — personal / team / enterprise, right-sized to risk.
Docs: Microsoft Purview · Azure AI Content Safety
How the Layers Connect

One prompt, six steps, every layer.

Concrete > abstract. Below is one end-to-end journey for a single question, the way it actually moves through the stack — with the specific Microsoft service handling each hop. Follow the numbered cards to see how the five layers and two rails work together.

Scenario: A regional sales manager opens Microsoft Teams and asks her custom "Pipeline Health" agent: "Summarize my top-10 at-risk deals this quarter and draft a follow-up email to each account owner."
1
The surface

Teams (Layer 5 — Apps & Surfaces) captures the prompt and identifies the user via Microsoft Entra ID.

Layer 5 · Identity rail
2
The agent

A custom Pipeline Health agent built in Copilot Studio (Layer 4) decomposes the question into tool calls and grounds intent.

Layer 4 · MCP tool calls
3
The gateway

Every model call passes through AI Gateway in APIM — llm-token-limit caps spend, Content Safety screens prompts, llm-emit-token-metric emits per-user telemetry.

Layer 3 · Governance rail
4
The model

Foundry routes the call to GPT-5.5 on reserved PTU capacity. Overflow lands on PAYGO via priority-based load balancing.

Layer 3 · PTU-first
5
The grounding

RAG retrieves CRM data from a Fabric semantic model over OneLake, filtered by Purview sensitivity labels so only deals she may see come back.

Layer 2 · Governance rail
6
The foundation

All of it runs on an Azure AI Landing Zone — private endpoints, managed identities, Azure Monitor capturing every span end-to-end.

Layer 1 · Identity rail
One prompt touches all five layers in under a second. The Identity rail ensures every hop has an owner; the Governance rail ensures every hop has policy. That is the Microsoft stack working as designed.

Glossary

The 16 words you'll see everywhere.

Open any term to see a one-paragraph definition, where it lives in the stack, and the official Microsoft Learn link. No deep theory — just the concept and the breadcrumb.

Agent Agent
A piece of software that uses an LLM plus tools, memory, and instructions to plan and execute work with some autonomy — from drafting an email to closing an expense claim. Lives in Layer 4 (Agentic Platform). Built in Copilot Studio (low-code) or Foundry Agent Service (pro-code).
See: Anatomy of an Agent
Agent MCP (Model Context Protocol)
The open wire format that lets LLM clients (Copilot, VS Code, Claude, etc.) discover and call tools. APIM can expose any REST API as an MCP server at /mcp and front external MCP servers with the same policies as REST.
See: AI Gateway in APIM · Docs: MCP server overview
Agent A2A (Agent2Agent)
Microsoft's protocol for agent-to-agent communication, letting multiple agents collaborate on a single workflow. Currently in preview as part of the March 2026 APIM AI Gateway release.
See: AI Gateway in APIM (timeline section)
Platform AI Gateway
A set of policies built into Azure API Management that make APIM AI-aware: token-aware throttling, semantic caching, content safety, OAuth 2.1 for MCP, PTU-first load balancing, and OpenTelemetry GenAI semantic conventions. Same gateway you already use for REST, taught new tricks for LLMs.
See: AI Gateway in APIM
AI RAG (Retrieval-Augmented Generation)
A pattern where the model is given relevant snippets of your data at inference time instead of being fine-tuned on it. The grounding step in our traced example. Built with Azure AI Search, Foundry IQ, or direct semantic-model queries.
See: IQ Platforms Guide
AI PTU (Provisioned Throughput Units)
Reserved model capacity priced per unit per month. Predictable cost and latency — the opposite of pay-as-you-go. AI Gateway's priority-based load balancing keeps PTU saturated first and overflows to PAYGO.
See: AI Gateway in APIM (Cost pillar)
Data OneLake
The single logical data lake underpinning Microsoft Fabric — one copy of your data, accessed by every Fabric engine. Sometimes called "the OneDrive for data". Lives in Layer 2 (Data Platform) and grounds nearly every enterprise agent.
Docs: OneLake overview
Data Foundry IQ
The retrieval layer for AI Foundry — multi-source grounding, indexing, and reranking under one API. Public Preview at time of writing.
See: IQ Platforms Guide
Data Fabric IQ
The semantic-model layer for Microsoft Fabric — verified, governed business semantics that grounding can trust. Public Preview.
See: IQ Platforms Guide
Data Work IQ
The M365 context layer — signals about the user's work patterns, calendar, files, and collaborators that personalize Copilot answers. Public Preview.
See: IQ Platforms Guide
Platform Microsoft Foundry
Microsoft's unified AI development platform (rebranded from Azure AI Foundry). 1,900+ models including GPT-5.5, GPT-5.4 family, Mistral Large 3, Llama, Phi. Fine-tuning, evaluation, observability, and the Agent Service all in one control plane.
Docs: Microsoft Foundry
Platform AI Landing Zone
The Microsoft reference architecture (GA Nov 17, 2025) for production AI workloads on Azure — networking, identity, security, observability, operating model. The Layer 1 foundation everything else depends on.
See: Azure AI Landing Zones
Platform Power Platform
Microsoft's low-code family in Layer 5 (Apps & Surfaces): Power Apps (canvas and model-driven apps), Power Automate (workflow automation), and Power Pages (external-facing sites). All three share Dataverse as their data backbone and can host or consume agents built in Copilot Studio — making it the fastest path from a business idea to a working app that uses the same models, tools, and governance as everything else in the stack.
Docs: Power Platform
Security Entra Agent ID
A non-human identity in Microsoft Entra ID, issued to an agent so every action it takes is attributable. Pairs with managed identities for backend access, removing API keys from the audit trail.
Docs: Microsoft Entra ID
Security Content Safety
Azure AI Content Safety — the multi-modal moderation service for prompts, completions, images, and AI-generated content. Wired into AI Gateway as a built-in policy so every prompt is screened before reaching the model.
Docs: Azure AI Content Safety
Security OAuth 2.1 for MCP
Native authentication for APIM-hosted MCP servers, aligned with MCP spec 2025-06-18. Released to GA in the March 2026 APIM AI Gateway release. Lets every MCP tool call carry a verifiable user identity end-to-end.
See: AI Gateway in APIM (Security pillar)
