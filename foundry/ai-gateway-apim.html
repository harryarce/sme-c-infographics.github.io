Why AI Gateway

Generic API platforms can't see tokens, prompts, or PTU buckets.

AI traffic looks like API traffic but behaves nothing like it. Tokens replace request counts as the unit of cost. Prompts and completions need moderation. PTU instances need preference. Agents and MCP clients negotiate on the wire. AI Gateway extends APIM with the AI-aware policies the new workload demands — without forcing a second control plane.

¢

Tokens are the new bill

Per-minute and per-quota caps on tokens — not requests. llm-token-limit pre-estimates prompt tokens so a runaway agent is blocked at the gateway, not at the model bill.

!

Prompts are user input

Every prompt deserves the same scrutiny as any other inbound payload. Native Azure AI Content Safety integration moderates LLM input — and supports patterns for output — before it reaches the model.

∞

PTU first, PAYGO when needed

Priority-based load balancing keeps PTU saturated and overflows to PAYGO. Circuit breakers honor each backend's Retry-After, so traffic resumes the second capacity is ready.

◇

Agents need a registry

MCP turns every API into a tool. Without a registry, agents can't discover them; without a gateway, you can't govern them. AI Gateway + API Center give you both.

The Capabilities

14 capabilities, organized as 5 pillars.

The Azure-Samples AI Gateway repo groups capabilities under the same Well-Architected pillars you already use elsewhere on Azure. Each policy below is named exactly as it appears in Microsoft Learn — the orange Preview tags mark features still in public preview at the time of writing.

Security

Govern, don't gatekeep.

  • Content safety — Azure AI Content Safety policy moderates LLM prompts
  • Managed identity to Foundry / Azure OpenAI — eliminates API keys
  • OAuth 2.0 / Entra ID via Credential Manager (inbound + outbound)
  • OAuth 2.1 native MCP auth aligned with MCP spec 2025-06-18 New
  • Private connectivity via APIM private endpoints & VNets

Reliability

One slow backend doesn't melt the system.

  • Backend load balancing — round-robin, weighted, priority, session-aware
  • Circuit breaker — dynamic trip honoring backend Retry-After
  • Token rate limit & quota — llm-token-limit, returns 429/403
  • Multi-region gateway — automatic capacity scaling, regional fan-out
  • Deployment-level token limits per model deployment 2026

Performance

Cache the answer, not just the request.

  • Semantic caching — llm-semantic-cache-store + -lookup
  • Backed by Azure Managed Redis or RediSearch-compatible cache
  • Vector proximity via Embeddings API — reuses prior completions
  • Model routing — direct latency-sensitive traffic to closer regions
  • PTU prioritization — keep premium capacity saturated first

Operations

Tokens you can see. Spend you can shape.

  • Token metrics — llm-emit-token-metric with custom dimensions
  • Prompt + completion logging to Application Insights
  • Built-in token-consumption dashboard in APIM
  • OpenTelemetry GenAI semantic conventions for agent APIs 2026
  • MCP runtime telemetry — latency, outcome, error details 2026

Cost

PTU first, PAYGO when you need it.

  • FinOps framework samples in the AI Gateway repo
  • Per-consumer token quotas — chargeback by subscription, IP, user
  • Semantic cache cuts tokens billed for repeated questions
  • Priority-based load balancing keeps paid PTU saturated
  • Backend pool across Foundry / AOAI / OpenAI-compatible

Robust & Observable

The two reasons platform teams adopt it.

Robustness is what keeps one app from starving another and one slow backend from melting the system. Observability is what lets you bill, audit, and debug what just happened. AI Gateway delivers both with the same policies and dashboards your APIM operators already know.

Why architectures are more robust

No more noisy-neighbor outages.

  • Multi-backend load balancing Round-robin, weighted, priority-based, session-aware distribution across Foundry, Azure OpenAI, and OpenAI-compatible endpoints.
  • PTU + PAYGO mixing Microsoft explicitly calls out priority modes that prefer PTU instances and overflow to PAYGO — your reserved capacity stays saturated, spillover lands on demand-priced.
  • Circuit breakers with dynamic trip duration APIM applies the value from the backend's Retry-After header — traffic resumes the moment the backend says it's ready, no fixed cooldowns wasting PTU.
  • Token-aware throttling llm-token-limit pre-computes prompt tokens and blocks at the gateway when the limit is already exceeded — no wasted backend calls, no noisy-neighbor outages.
  • Multi-region gateway deployment APIM scales gateway capacity automatically and supports regional gateway addition; pair with backends in matching regions for geo-resilience.
  • Deployment-level token limits March 2026 release — caps applied per model deployment, not just per consumer. Prevents one rogue model deployment from taking down the bucket.
One app cannot starve another. One backend going slow does not melt the system. PTU spend stays inside the bucket you paid for.

Why architectures are more observable

Every prompt, completion, and token has an owner.

  • llm-emit-token-metric Emits token consumption to Azure Monitor with custom dimensions — Microsoft's docs show Client IP, API ID, and User-ID-from-header. Slice token spend by consumer, model, operation, or any policy expression.
  • Prompt + completion logging Diagnostic settings stream prompts and completions to Application Insights for billing, auditing, and troubleshooting.
  • Built-in token-consumption dashboard APIM ships a default monitoring dashboard so you don't have to wire one up to start tracking spend.
  • OpenTelemetry GenAI semantic conventions Agent APIs flow agent name, agent ID, and provider name into App Insights as standards-aligned OTel attributes. March 2026
  • MCP runtime telemetry Tool-invocation latency, request outcomes, and error details for every MCP call — observe agent-tool traffic the same way you observe REST APIs.
  • Managed identity to backends Removes secret-rotation noise from the audit trail — every backend call traces to a workload identity, not a vault entry.
Every prompt, completion, and token has an owner, a metric, and a dashboard.
AI Gateway + API Center

A registry for the agent era.

AI Gateway is the runtime control plane. Azure API Center is the organizational catalog. Together they give every developer a self-service way to find, govern, and call MCP tools — and every platform team one place to apply policy, identity, and rate limits to AI traffic.

Runtime

Azure API Management — AI Gateway

Hosts and fronts MCP servers. Applies policy, identity, rate-limit, content safety, and logging to every tool call.

  • Expose any APIM-managed REST API as an MCP server at /mcp
  • Front existing MCP servers (Logic Apps, Functions, LangChain, custom) via pass-through
  • OAuth 2.1 / Entra ID / API keys / IP filtering at the edge
  • Rate-limit-by-key on Mcp-Session-Id
  • Credential Manager outbound token injection (now with Key Vault references)

⚠ Important caveats — don't overclaim

  • For REST-derived MCP servers in APIM, only MCP tools are supported — resources and prompts are not supported today, and MCP capabilities are not supported in workspaces.
  • MCP support in v2 SKUs (Basic v2 / Standard v2 / Premium v2) is in public preview as of Aug 5, 2025.
  • The MCP tool limit per server was previously hardcoded at 20 — removed in the March 2026 release. Now bounded by the APIM SKU operation limit.
  • A2A support and AI Gateway in Microsoft Foundry are preview. Show a "Preview" badge wherever your design depends on them.
