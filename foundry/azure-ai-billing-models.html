The Model Map

Seven Billing Models — One Ecosystem

Azure AI billing is not a single meter. It maps to workload patterns. Matching the right model to the right workload is the single highest-leverage cost decision you will make.

💳 Standard / Pay-As-You-Go

Pay-As-You-Go (Standard)

Billed per input token and output token actually consumed. No commitment, no minimum. Cost scales directly with usage — zero idle charge.

Use when: Pilots, variable traffic, early-stage, unknown demand
Forecast: Requests × avg (input + output) tokens per request
Watch out: Output tokens cost 4–8× input; rate limits at scale
⚡ Priority Processing

Priority Processing

Pay-per-token, same as Standard, but on a priority tier with lower, more consistent latency. Higher per-token cost for the performance guarantee.

Use when: Latency-sensitive, not yet ready to provision PTUs
Forecast: Same as PAYG; apply the priority rate premium
Watch out: Higher unit cost than Standard; validate whether PTU would be more economical
🟢 Global Batch (~50% off)

Global Batch

Asynchronous model processing at approximately 50% less than Global Standard. Uses a separate token quota so batch jobs never compete with interactive workloads. Completions returned within 24 hours.

Use when: Offline summarization, bulk classification, document processing
Forecast: Jobs × avg tokens per job; batch quota is independent
Watch out: Not suitable for real-time user interactions
🖥️ Managed Compute

Managed Compute (GPU/hr)

Dedicated GPU infrastructure for open-source and custom models. Pay per compute-hour for the GPU you choose — A100, H100, H200, or MI300 — regardless of actual token throughput.

Use when: Open-source models, fine-tuned custom models, GPU-bound workloads
Forecast: GPU-hours deployed × hourly rate per GPU type
Watch out: Idle GPU still billed; size carefully for utilization
⏱ PTU Hourly

Provisioned Throughput (PTU) — Hourly

Dedicated model-processing capacity measured in Provisioned Throughput Units. Billed hourly per deployed PTU count — whether or not requests are being handled. PTUs are model-independent: the same pool deploys any supported model.

Use when: Predictable, latency-sensitive production traffic
Forecast: PTUs deployed × hours deployed × PTU hourly rate
Watch out: Billing stops only on deletion; validate utilization before committing
📅 PTU Reservation

PTU Azure Reservation

A commercial reservation that covers matching PTU hourly usage at a significantly discounted rate. Monthly and annual reservations available. Provides capacity assurance for sustained production workloads.

Use when: Steady PTU usage expected to run continuously; utilization proven via hourly
Forecast: Reserved PTUs × expected coverage × reservation rate
Watch out: Reserve only after hourly PTU utilization is proven stable; capacity risk on scale-down/up
🏦 Agent Prepurchase Plan / ACUs

Agent Prepurchase Plan (ACUs)

A commercial pool that pays eligible retail AI usage across Microsoft Foundry, Copilot Studio, Microsoft Fabric, and GitHub. ACUs are not the technical meter — they optimize cost after eligible usage is proven.

Use when: Broad AI adoption across multiple Microsoft platforms at scale
Forecast: Expected eligible retail usage for the one-year term
Watch out: Apply only after credible, measurable cross-platform usage is established
The Fundamental Decision

Pay-Per-Token vs. Provisioned Throughput

Every Foundry model deployment bills one of two ways. Getting this decision right determines most of your bill — everything else is tuning.

Pay-Per-Token (PAYG & Batch)

💰 Consume what you use

Charged for input and output tokens actually consumed. No idle cost, no minimum capacity. Cost scales directly with usage. Rate limits can apply at high volume.

Traffic patternBest fit
Spiky, unpredictable, early-stageStandard PAYG
Large async jobs, no latency needGlobal Batch (~50% off)
Latency-sensitive, not yet PTU-readyPriority Processing
Provisioned Throughput (PTU)

🎯 Reserve what you need

Dedicated model-processing capacity billed hourly per deployed PTU — regardless of actual requests. Predictable performance, no rate limits, model-independent quota pool.

Traffic patternBest fit
Steady, high, latency-sensitivePTU Hourly (validate first)
Continuous production, proven usagePTU Reservation (discount)
Open-source / custom GPU modelManaged Compute (GPU/hr)
The advanced insight: A mature production setup runs all three simultaneously — PTU for the latency-critical interactive path, Standard PAYG for spillover and development, and Batch for overnight bulk work. Each workload priced on the model that fits it, not one model for everything.

Token Economics

Two Numbers That Drive Your Model Bill

Every Azure AI Foundry model call is billed on two meters. Understanding the ratio between them is the fastest way to find avoidable cost.

$

Input tokens — the controllable side

Everything sent to the model: system prompt, user message, retrieved knowledge-base chunks, and conversation history. Directly controllable through prompt design and retrieval architecture.

A RAG prompt with system instructions, retrieved chunks, and chat history can reach 6,000–10,000 tokens before the user submits a question. Every chunk you retrieve costs input tokens.

4–8×

Output tokens — the expensive side

What the model generates in response. Generated sequentially token-by-token, not in parallel — which is why output tokens cost 4–8× more than input tokens depending on the model.

Verbose response instructions directly drive output cost. Set “max_tokens” limits, use structured outputs, and avoid open-ended completion patterns in high-volume scenarios.

What the Token Calculator Misses

Six Hidden Cost Categories That Inflate Bills

Microsoft’s official pricing calculator covers the token portion accurately. These six categories consistently push real deployment bills 15–40% above the token estimate — forecast each independently.

Search Layer

Azure AI Search / RAG Infrastructure

Indexing costs, document processing units, query volume, semantic ranker charges, and index storage. High query volume with semantic ranking can be a significant cost center at scale.

⚠ Often the #2 line item in RAG apps
Fine-Tuning

Fine-Tuned Model Hosting

Training cost (per million tokens) + hosting cost per deployment-hour + inference usage. The hosting clock runs continuously once an endpoint is deployed, regardless of traffic.

⚠ Hosting billed 24/7; delete idle fine-tune endpoints
Storage

Blob Storage & Data

Grounding data, knowledge base blobs, batch job inputs and outputs, model artefacts, and evaluation datasets all accumulate in Azure Storage and are billed for reads, writes, and capacity.

📊 Scale with corpus size
Observability

Monitoring & Logging

Azure Monitor ingestion, Log Analytics workspace, Application Insights, token usage logging, and cost management alerts. Every trace and metric exported has an ingestion and retention cost.

📊 Sample aggressively for low-value traces
Networking

Networking & Data Residency

Data Zone and Regional deployments carry a premium over Global Standard for data-residency compliance. Cross-region egress charges apply when services are not co-located.

⚠ Co-locate AI Search + Foundry in same region
Infrastructure

Support & Infrastructure Overhead

Azure support plan costs, VNet integration, private endpoints, APIM gateway for multi-app PTU sharing, and managed identity overhead all contribute at production scale.

📊 Plan for ~5–10% infrastructure overhead
Bottom line: A real production AI bill = model tokens + Azure AI Search + fine-tune hosting + blob storage + monitoring + networking + infrastructure. The Forecasting Framework

Five Steps to a Responsible Forecast

Start with workload mapping, instrument the pilot, build scenarios with real variance, optimize before committing, then select commercial models based on proven demand — not projected optimism.

1

Map Workloads

Group use cases by pattern: chat, agentic workflow, batch, retrieval, document, voice, multimodal. Identify expected users, requests, latency targets, data sources.

2

Instrument Pilot

Capture real consumption: tokens per request, tool calls, retrieval counts, pages processed, audio minutes, error retries. Tag every resource from day one.

3

Build Scenarios

Model average, p95, peak-day, and growth cases. Include concurrency, seasonality, SLA expectations, and the mix of interactive vs offline requests.

4

Optimize First

Reduce avoidable consumption before commercial commitments. Optimize prompts, route models by complexity, cache aggressively, and move eligible offline jobs to Batch.

5

Select Commercial Model

Apply PTU reservations or ACUs only to proven demand. Review utilization against reservation coverage. Validate Azure Pricing Calculator estimates with actual metered spend.

Total AI bill = model tokens (in + out) + Azure AI Search (index + query + ranker) + fine-tune (training + hosting + inference) + agent runtime (compute + memory + tools) + storage + monitoring + networking Commercial optimization = PTU reservations + ACU prepurchase + agreement terms

Cost Governance

Six Spend Controls Every Deployment Should Have

Governance is not overhead — it is what makes AI spend predictable. Apply these controls before your first production deployment, not after the first surprise bill.

Budget Alerts in Azure Cost Management

Set budgets per workload, per team, and per subscription. Configure multi-level alerts (50%, 80%, 100% forecast). Review weekly during the first month of any new deployment.

TPM Limits on Every Deployment

Set Tokens-Per-Minute limits on each model deployment. Prevents runaway cost from bugs, prompt injection, or unexpected traffic spikes. Validate limits against SLA before go-live.

Tag Every Resource at Creation

Apply environment, workload, team, and cost-center tags to every Foundry resource at creation time. Retroactive tagging is expensive. Tags enable cost attribution and showback reports.

Size for Baseline; Spill to PAYG

When moving to PTU, size provisioned capacity for the baseline load — not peak. Route overflow to Standard PAYG. Prevents over-provisioning and the cost of idle PTU capacity during off-peak hours.

Validate Model Selection Before Scaling

Run smaller, cheaper models on tasks that do not require frontier capability. Phi-4-mini for simple classification; GPT-4o for complex reasoning. Model routing is often the highest-ROI optimization.

Use an AI Gateway for Multi-App PTU Sharing

When multiple applications share PTU capacity, route through Azure API Management. The gateway normalizes token rates, enforces per-app quotas, and surfaces per-consumer cost visibility.
