The Ten Organs

Every agent is built from these parts.

Ship with all ten and the organism holds. Skip one and it breaks — always in the exact way the missing organ would have prevented.

1. Instructions

the heart — purpose, persona, constraints

The agent's constitution — who it is, what it exists to do, and what it must never do. Every other organ serves this one.

  • Role & persona — "You are a Tier-1 support triage agent for Contoso."
  • Goals & success criteria — a precise definition of "done"
  • Hard constraints — policy guardrails, compliance boundaries, refusal rules
  • Tone & output shape — format, voice, length, audience
Without itThe agent drifts. Same prompt, wildly different outputs, no way to audit why — and no way to fix it.

2. Model

the brain — reasoning engine

The LLM that reads, reasons, and writes. Not one size fits all — pick for the job: reasoning depth vs. latency vs. cost vs. context window.

  • Frontier — GPT-5.4 / GPT-5.5, Claude Opus, Gemini Ultra for deep reasoning and complex multi-step tasks
  • Workhorse — GPT-5.4-mini / GPT-4.1 for mainstream tasks at a fraction of frontier cost
  • Small & local — Phi-4 and nano-class models for edge, low-latency, on-device scenarios
  • Fine-tuned — when your domain's language matters more than raw capability
Without the right oneYou're either overpaying 10× for trivial tasks, or watching your agent collapse on the ones that actually matter.

3. Knowledge & Context

the senses — what it sees and hears

Everything the agent perceives before it reasons — retrieval results, grounding sources, real-time signals, and the user's current situation. Context is the cure for hallucination.

  • Retrieval — vector + keyword search over your enterprise corpus (Foundry IQ, Azure AI Search)
  • Grounding sources — SharePoint, OneDrive, Fabric IQ, Work IQ, web
  • Federated knowledge — live queries to systems of record across boundaries
  • Session context — user intent, recent turns, attachments, activity signals
Without itConfident hallucination. The agent invents policies, invents customers, invents facts — and delivers them with perfect grammar.

4. Tools

the hands — how it acts on the world

Functions, APIs, MCP servers, and A2A endpoints the agent can invoke. This is the line between "talks about work" and "does work."

  • Function calls — typed, schema-validated capabilities with structured I/O
  • MCP servers — portable, reusable tool catalogs via the Model Context Protocol
  • Connectors — Logic Apps, Microsoft Graph, Power Platform (~1,400+ connectors)
  • Code interpreter — sandboxed compute for ad-hoc analysis and transformation
Without themA chatbot. Can eloquently describe the fix. Can't apply it.

5. Skills

muscle memory — trained procedures

Named, reusable playbooks that bundle instructions, tool sequences, and examples for a recurring job. Not code — packaged expertise.

  • Composable — one agent, many skills; one skill, shared across many agents
  • Conditionally loaded — invoked only when the trigger fires, keeping context lean
  • Expert-encoded — your best operators' patterns, distilled and versioned
  • Evolvable — improve the skill without rewriting the agent
Without themEvery task is re-derived from first principles. Brittle, inconsistent, and impossible to scale across a fleet of agents.

6. Memory

the hippocampus — what it remembers

State that outlives a single turn — short-term scratch, long-term facts about the user, episodic traces of past sessions. Memory is what makes an agent feel like a colleague, not a stranger.

  • Short-term — the working conversation window and scratchpad
  • Long-term — user preferences, prior decisions, domain vocabulary
  • Episodic — "last time we discussed X, we decided Y"
  • Shared — team-level memory that spans agents and sessions
Without itPermanent amnesia. Users repeat themselves every session. Personalization is a fantasy.

7. Guardrails

the ribcage — protecting what matters

Input, output, and runtime safety — not bolted on after launch, but woven into every call. The ribcage doesn't slow the heart; it protects it.

  • Input shields — prompt injection detection, jailbreak prevention
  • Output filters — PII redaction, toxicity detection, groundedness checks
  • Policy checks — tool-use allowlists, data-boundary enforcement, Purview labels
  • Human-in-the-loop — approval gates for high-stakes, irreversible actions
Without themOne jailbreak or prompt injection exfiltrates data, books travel, or sends the wrong email — and you find out from a customer.

8. Orchestration & Planning

the nervous system — how it decides

The control loop that turns "answer this" into "plan, act, observe, re-plan." Single-agent, multi-agent, hand-offs, parallel branches — the nervous system that makes everything else move.

  • ReAct loop — reason → act → observe → repeat until done
  • Plan-and-execute — decompose the problem first, then run the plan
  • Multi-agent — specialist agents coordinated by a router or supervisor via A2A
  • Hand-offs — escalate to a peer agent or a human when the task demands it
Without itOne-shot answers only. The agent can't recover from its own mistakes, can't chain actions, and can't learn mid-task.

9. Identity & Permissions

the badge — who the agent acts as

Agents are first-class workforce identities — they authenticate, inherit permissions, and leave audit trails, just like employees. No badge, no access.

  • Entra Agent ID — a dedicated agent identity in Microsoft Entra, not a shared service key
  • Delegated access — act on behalf of the signed-in user, scoped to their permissions
  • Least-privilege tools — only the APIs and data the agent's role requires
  • Audit trail — every action attributable to this specific agent identity
Without itOver-privileged service principals with God-mode access. Compliance can't prove what the agent did — or didn't do.

10. Observability & Evals

the stethoscope — how you know it's healthy

Tracing, logging, evals, and telemetry — the difference between "it works on my demo" and "it works at scale, under adversarial conditions, on day 90."

  • Traces — every step, tool call, token count, and latency, end-to-end
  • Evals — accuracy, groundedness, safety, and task-completion metrics
  • Red-teaming — adversarial testing before every production release
  • Production monitoring — drift detection, cost tracking, user satisfaction signals
Without itFlying blind. Regressions go unnoticed until a customer escalates on Teams — and by then, trust is gone.

Diagnostic

Is your agent fully formed?

Score yourself honestly. If you can't confidently check all ten, what you have is an agent-shaped prototype — not a production system.

The Ten-Point Exam

One checkmark per organ. No partial credit.

✓
Instructions — a versioned system prompt with persona, goals, hard constraints, and explicit refusal rules.
✓
Model — a deliberate choice with cost / latency / capability tradeoffs documented — not "whatever was default."
✓
Knowledge — grounded retrieval over a curated, governed corpus — not the model's pre-training data.
✓
Tools — typed, schema-validated functions via MCP or OpenAPI — not free-form shell or raw HTTP.
✓
Skills — named, reusable playbooks for recurring tasks, conditionally loaded on demand.
✓
Memory — short-term working state plus at least one durable memory surface (user, team, or shared).
✓
Guardrails — input shields + output filters + tool policy checks, with human-in-the-loop for high-stakes writes.
✓
Orchestration — a real control loop (ReAct / plan-execute / multi-agent), not one-shot prompt-then-respond.
✓
Identity — its own Entra Agent ID with least-privilege tool permissions and a full audit trail.
✓
Observability — end-to-end traces + an eval suite that runs on every change, not just before launch.
Pathology

The six ways agents fail.

Every failure mode maps to a missing or malformed organ. Use this list in post-mortems — the root cause is almost always one of these, hiding behind a more dramatic symptom.

Anti-pattern 1

The Clever Prompt

All instructions, no tools, no retrieval. "Prompt engineering" treated as the entire architecture.

→ Add knowledge and tools. Clever prompts scale to demos. Systems scale to production.

Anti-pattern 2

The Tool Hoarder

Forty tools attached, no skills, no orchestration. The model drowns trying to pick the right one — and often picks wrong.

→ Bundle tools into named skills. Load skills conditionally. Less choice = better decisions.

Anti-pattern 3

The Ungrounded Oracle

Frontier model, slick persona, zero retrieval. Confidently wrong about your own business, your own customers, your own policies.

→ Ground every factual claim in retrieved knowledge. Teach the agent to say "I don't know."

Anti-pattern 4

The Amnesiac

No memory surface. Users explain themselves and their context from scratch — every single session.

→ Add short-term summarization and at least one durable memory surface for user preferences.

Anti-pattern 5

The Unleashed Intern

Broad tool access, no guardrails, no identity scoping. One prompt injection away from catastrophe.

→ Entra Agent ID + least-privilege tools + input/output shields + human approval gates.

Anti-pattern 6

The Black Box

No traces, no evals, no telemetry. Every regression is a surprise; every incident is a forensic archaeology dig.

→ Emit traces by default. Run evals on every change — not just before launch day.

The Ladder

From skeleton to athlete.

No agent ships fully formed. They grow — organ by organ, rung by rung. Use these levels to locate your agent today, and see what it takes to reach the next stage.

Rung 1

Skeleton

  • Instructions only — a system prompt
  • Single model call, no loop
  • No tools, no retrieval, no memory
  • Impressive in demos. Dangerous in production.
Rung 2

Functional

  • + Knowledge grounding (RAG / Foundry IQ)
  • + A curated set of tools via MCP
  • + Basic input/output guardrails
  • Does real work. Breaks under adversarial pressure.
Rung 3

Production-ready

  • + Skills catalog with conditional loading
  • + Memory surfaces (short-term + long-term)
  • + Entra Agent ID with least-privilege scoping
  • + End-to-end traces and evals on every change
Rung 4

Athletic

  • + Multi-agent orchestration via A2A protocol
  • + Shared team memory across agent fleets
  • + Adversarial red-team suite before every release
  • + Continuous eval and drift detection in production
