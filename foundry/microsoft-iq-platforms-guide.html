Deep Dive

The Four IQ Platforms

Each capability contributes a distinct kind of context. Together, they give agents a fuller understanding of how people work, how the business operates, what the organization knows, and what is happening on the web.

Workplace intelligence
💼

Work IQ

The intelligence layer behind Microsoft 365 — context, memory, and action for every agent that lives where work happens

What it is

M365 Context MCP Servers Memory
  • A workplace intelligence layer that combines Microsoft 365 data with contextual understanding so agents can reason over work and take action
  • Provides live understanding of people, collaboration, and workflows
  • Supports Agent-to-Agent (A2A), Model Context Protocol (MCP), and REST integration patterns
  • Reasons over email, meetings, calendar data, OneDrive and SharePoint documents, Teams messages, people, Planner plans, and enterprise search results

How it works

  • Choose A2A for agent delegation, MCP for tool-based context, or REST for service-hosted agents and orchestrators
  • Requests run in the context of the signed-in user through Microsoft Entra ID delegated authentication
  • Microsoft 365 permissions, sensitivity labels, compliance policies, and permission trimming are enforced automatically
  • Context is retrieved dynamically when needed, without maintaining custom vector stores or data-sync jobs
  • Application-only authentication is not supported

When to use it

  • Agentic and AI-powered applications that need secure Microsoft 365 grounding
  • Personal productivity agents: email triage, meeting preparation, and document drafting
  • "Summarize my week across Teams, email, and calendar"
  • Agents that need activity context — who's collaborating with whom on what
  • Multi-agent systems, IDE and CLI assistants, or service-hosted applications

Business intelligence context

Fabric IQ

Your business vocabulary, made machine-readable — the semantic foundation that turns data platforms into intelligence platforms

What it is

Business Ontology Semantic Graph Data Agents
  • Part of Microsoft IQ that provides context on the live state of your business
  • Grounded in analytical, real-time, and operational data about your business
  • Models business entities and their relationships, properties, actions, and rules
  • Helps people and agents reason and make decisions in terms of business concepts and objectives

How it works

  • Define entities (Customer, Order, Asset) and their measures once in the Ontology item — bootstrap from existing Power BI semantic models in clicks
  • Data Agents answer natural-language analytics questions grounded in your data model and business definitions
  • Operations Agents monitor real-time data, detect anomalies, and trigger workflows autonomously within defined business rules
  • Queries cascade through Graph + OneLake with row/column-level security preserved on every hop
  • Plugs into Foundry IQ as a first-class knowledge source — seamless cross-platform retrieval

When to use it

  • Agents that need to reason over structured business data: revenue, inventory, customers, projects
  • Analytic / BI copilots with correct terminology and metric definitions
  • Cross-domain questions: "Which customers with open support tickets also have renewals this quarter?"
  • Operational automation: auto-detect inventory risks, trigger reorder workflows
  • When you're already on Microsoft Fabric — Fabric IQ is included in your SKU

Managed enterprise knowledge
✨

Foundry IQ

One retrieval endpoint for every agent — ubiquitous, governed knowledge across your entire ecosystem

What it is

Unified Retrieval Agentic Planning Context Engineering
  • A managed knowledge layer for enterprise data that creates configurable, multi-source knowledge bases
  • Built on Azure AI Search indexing and retrieval infrastructure, with agentic retrieval for complex questions
  • Core concepts: Knowledge Bases (resources that orchestrate retrieval) and Knowledge Sources (connections to indexed or remote content)
  • Three retrieval reasoning tiers: minimal, low, medium — dial the knob between speed and depth per query

How it works

  • Connect a knowledge base to one or more agents and add the knowledge sources that it should query
  • Agentic retrieval uses an optional Azure OpenAI model to plan queries, select sources, run parallel searches, and aggregate results
  • Supports keyword, vector, and hybrid queries over indexed and remote sources, including Azure Blob Storage, SharePoint, OneLake, and public web data
  • Returns extractive data with citations so agents can reason over raw content and trace answers to source documents
  • Synchronizes supported ACLs, honors Microsoft Purview sensitivity labels, and enforces permissions at query time

When to use it

  • Any agent that needs knowledge from more than one source (the vast majority)
  • Enterprise RAG patterns — replace hand-written retrieval orchestration
  • Multi-domain agents: combines M365 context (Work IQ) + business data (Fabric IQ) + domain docs
  • When you need a single governed, auditable retrieval layer with Purview monitoring
  • Pro-code agents on Foundry or low-code agents in Copilot Studio

Fresh web intelligence
🌐

Web IQ

Search built for the AI era — model-agnostic, MCP-native grounding from the open web for every agent

What it is

Web Grounding MCP-Native Model-Agnostic
  • A Microsoft IQ capability that provides AI systems and agents with fresh, real-world intelligence from across the web
  • Adds current public information to the organizational context supplied by the other Microsoft IQ capabilities
  • Complements Work IQ, Fabric IQ, and Foundry IQ when an agent needs information beyond enterprise data
  • Use it to ground research and agent experiences in timely information from the web

How it works

  • Use fresh web intelligence alongside the work, business, and institutional context available through the other IQ capabilities
  • Choose the sources that fit the agent's task and the information its users need
  • Use primary sources and citations appropriate to your agent's experience and governance requirements
  • Validate the current Web IQ integration options in the official product documentation before implementation

When to use it

  • Any agent that needs current, public information — news, prices, regulations, product specs
  • Research, competitive intelligence, market scanning, and customer-due-diligence flows
  • Customer-facing assistants that must answer beyond what’s in your enterprise corpus
  • To replace ad-hoc third-party web-search APIs with a single, governed Microsoft surface

Building a Smarter Agent with Microsoft IQ

Start with the agent's task, then add the Microsoft IQ capabilities that supply the context it needs: work signals, business state, institutional knowledge, and fresh web intelligence.

1
Define the Agent

Build in Copilot Studio (no/low-code) or with the Foundry Agent SDK and Agent Framework (pro-code). Choose your model.

2
Choose the Context

Map the task to the context it needs: work activity, business data, knowledge bases, web intelligence, or a combination.

3
Attach Work IQ

Use A2A, MCP, or REST to let the agent reason over permission-trimmed Microsoft 365 work context.

4
Attach Fabric IQ

Connect business entities, relationships, properties, actions, and rules to ground decisions in the state of the business.

5
Add Knowledge & Web Context

Use Foundry IQ knowledge bases for curated institutional knowledge and Web IQ when fresh web intelligence is needed.
