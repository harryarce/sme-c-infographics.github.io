Work IQ

The intelligence layer behind Microsoft 365 — context, memory, and action for every agent that lives where work happens

What it is

M365 Context MCP Servers Memory
  • The intelligence layer that grounds M365 Copilot and your agents in real-time organizational context
  • Three layers: Data (who, what, where), Memory (persistent personal context), Inference (reasoning + action over work signals)
  • 9+ MCP servers: Copilot, Calendar, Mail, SharePoint, OneDrive, Teams, User, Word, plus Dataverse & Dynamics 365
  • Available via Copilot Studio, Agent 365, Microsoft Foundry, VS Code, or standalone CLI / GitHub Copilot plugin

How it works

  • Agent calls a Work IQ MCP server (e.g., Work IQ Mail, Work IQ SharePoint)
  • MCP server queries the Microsoft Graph on behalf of the signed-in user — scoped, auditable, compliant
  • Delegated Entra ID permissions enforce user-scoped access on every call
  • Memory layer builds persistent understanding of how people and teams work across sessions
  • Returns structured, citation-ready results the agent can reason and act on

When to use it

  • Personal productivity agents: email triage, meeting prep, document drafting
  • "Summarize my week across Teams, email, and calendar"
  • Agents that need activity context — who's collaborating with whom on what
  • Shipping into Teams, Outlook, or M365 Copilot surfaces
  • When you already have M365 Copilot licenses

Fabric IQ

Your business vocabulary, made machine-readable — the semantic foundation that turns data platforms into intelligence platforms

What it is

Business Ontology Semantic Graph Data Agents
  • New Microsoft Fabric workload that elevates Fabric from a data platform into a unified intelligence platform
  • Six core items: Ontology, Digital Twin Builder, Graph, Data Agent, Operations Agent, Semantic Models (Power BI)
  • Ontology captures your business entities, relationships, rules, and KPIs — so agents speak your language, not database jargon
  • Native Graph engine reasons over connected enterprise data in OneLake with multi-hop traversal

How it works

  • Define entities (Customer, Order, Asset) and their measures once in the Ontology item — bootstrap from existing Power BI semantic models in clicks
  • Data Agents answer natural-language analytics questions grounded in your data model and business definitions
  • Operations Agents monitor real-time data, detect anomalies, and trigger workflows autonomously within defined business rules
  • Queries cascade through Graph + OneLake with row/column-level security preserved on every hop
  • Plugs into Foundry IQ as a first-class knowledge source — seamless cross-platform retrieval

When to use it

  • Agents that need to reason over structured business data: revenue, inventory, customers, projects
  • Analytic / BI copilots with correct terminology and metric definitions
  • Cross-domain questions: "Which customers with open support tickets also have renewals this quarter?"
  • Operational automation: auto-detect inventory risks, trigger reorder workflows
  • When you're already on Microsoft Fabric — Fabric IQ is included in your SKU

Foundry IQ

One retrieval endpoint for every agent — ubiquitous, governed knowledge across your entire ecosystem

What it is

Unified Retrieval Agentic Planning Context Engineering
  • Context engineering platform in Microsoft Foundry — a single retrieval endpoint that spans every knowledge source your agent needs
  • Built on Azure AI Search, extends it with agentic retrieval, LLM-powered query planning, and intelligent ranking
  • Core concepts: Knowledge Bases (curated collections of sources) and Knowledge Sources (indexed files, Fabric IQ, Work IQ, web, SharePoint, and custom)
  • Three retrieval reasoning tiers: minimal, low, medium — dial the knob between speed and depth per query

How it works

  • Agent makes one call to a Foundry IQ knowledge base — no hand-written retrieval orchestration
  • A query-planning LLM (GPT-4.1, GPT-5, GPT-5.4, or GPT-5.5) decomposes the question into targeted sub-queries
  • Sub-queries fan out in parallel across indexed content, Fabric IQ, Work IQ, web, and custom remote sources
  • Results are ranked, deduplicated, and consolidated with full citations — ready for the agent to reason over
  • Entra ID, Purview, and data-source ACLs enforced on every hop — zero-trust retrieval by design

When to use it

  • Any agent that needs knowledge from more than one source (the vast majority)
  • Enterprise RAG patterns — replace hand-written retrieval orchestration
  • Multi-domain agents: combines M365 context (Work IQ) + business data (Fabric IQ) + domain docs
  • When you need a single governed, auditable retrieval layer with Purview monitoring
  • Pro-code agents on Foundry or low-code agents in Copilot Studio

End-to-End

Building a Smarter Agent with IQ

The canonical pattern: define your agent, front it with Foundry IQ as the unified knowledge layer, then wire in Work IQ and Fabric IQ to light up the full spectrum of organizational intelligence.

1
Define the Agent

Build in Copilot Studio (no/low-code) or with the Foundry Agent SDK and Agent Framework (pro-code). Choose your model.

2
Create a Foundry IQ Knowledge Base

Start with one knowledge base. Add sources: docs, SharePoint, Work IQ, Fabric IQ.

3
Attach Work IQ

Plug in MCP servers for Mail, Calendar, SharePoint — so the agent sees user activity context.

4
Attach Fabric IQ

Register Fabric IQ ontology + Data Agents to ground answers in business metrics and semantics.

5
Govern & Ship

Apply Purview policies, set retrieval reasoning tier, deploy to Teams, web, or your app.
