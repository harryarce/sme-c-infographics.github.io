The Seven Pillars

The design areas every AI Landing Zone gets right.

The AI ALZ formalizes design areas straight from the Cloud Adoption Framework and the Well-Architected Framework's AI workload pillars. Each one has been pre-decided, pre-wired, and pre-validated — so your team doesn't have to.

Networking

no public surface area by default

Hub-spoke VNet topology with private endpoints to every PaaS service and private DNS zones — your AI estate is invisible to the open internet.

  • NSGs with default-deny baseline
  • Private endpoints for Foundry, AI Search, Storage, Key Vault, Cosmos DB
  • Private DNS zones, hub-managed
  • Optional Bastion + jumphost for operator access

Identity & Access

a VIN number for every agent

Microsoft Entra ID at the core, managed identities everywhere, and Microsoft Entra Agent ID giving every AI agent a first-class, governable identity.

  • Managed identities — no shared service keys
  • RBAC scoped least-privilege per resource
  • Entra Agent ID for agent governance & audit
  • Conditional Access policies enforced

Security

baked in, not bolted on

Defender for Cloud's AI Security Posture Management plus Content Safety, Prompt Shields, and Key Vault — runtime threats blocked before they reach your model.

  • Defender for Cloud + AI SPM
  • Content Safety + Prompt Shields plumbed into Foundry
  • Key Vault for secrets & customer-managed keys
  • GitHub Advanced Security hooks for the pipeline

Governance & Compliance

audit-ready from day one

Azure Policy, Microsoft Purview classification & DLP for AI, and resource locks — compliance posture inherited, not retrofitted.

  • Azure Policy assignments at landing-zone scope
  • Purview for data classification & AI DLP
  • Responsible AI guardrails as defaults
  • Resource locks on critical infrastructure

AI Platform Services

the factory floor, pre-installed

Microsoft Foundry account, projects, and Agent Service — wired to Azure AI Search, Cosmos DB, and Storage. Build the AI; skip the wiring.

  • Foundry Account + Projects + Agent Service
  • Azure AI Search for RAG retrieval
  • Cosmos DB for agent state & memory
  • Storage Account for blobs & grounding sources

Observability

deployed before workloads

Log Analytics + Application Insights stand up first — so the moment a workload lands, every prompt, tool call, and span is already being captured.

  • Single Log Analytics workspace, all signals
  • App Insights wired to every service
  • OpenTelemetry-compatible agent tracing
  • Cost dashboards out of the box

DevOps & IaC

verified modules, not snowflakes

Bicep, Terraform, and Portal deployments — all built on Azure Verified Modules. Every resource follows Microsoft's curated standard, every change is reviewed, every deployment is repeatable.

  • Bicep for ARM-native teams
  • Terraform for multi-cloud teams
  • Azure Portal for click-through provisioning
  • GitHub Actions / Azure DevOps templates

Maturity Progression

Sandbox → Enterprise → Regulated.

The same Landing Zone scales with you. Start with a public-endpoint sandbox to prove the value, lock it down for enterprise rollout, then add the controls that satisfy your most regulated business unit.

Stage 1

Sandbox

Public endpoints · minimal services · move fast

  • Foundry account + project + Agent Service
  • Public endpoints, basic RBAC
  • Quick proof-of-concept in a single subscription
  • Ideal for innovation teams & demos
Stage 2

Enterprise

Private networking · BYOR · audit-ready

  • VNet isolation & private endpoints everywhere
  • Key Vault, Cosmos DB, AI Search wired in
  • Log Analytics + App Insights from day one
  • Defender for Cloud + Azure Policy enforced
  • Entra Agent ID for every agent
Stage 3

Regulated

Sovereign-ready · zero-trust · audit-complete

  • APIM AI Gateway pattern for centralized control
  • Purview labels & DLP for AI in production
  • Customer-managed keys (CMK) end-to-end
  • Azure Government / Sovereign Cloud-ready
  • Foundry Control Plane for full audit trail
