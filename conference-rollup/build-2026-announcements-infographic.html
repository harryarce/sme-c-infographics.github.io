02

AI Models

3 ANNOUNCEMENTS

Microsoft's in-house MAI family expands to seven new models, with reinforcement-learning customization and broader third-party model access on Foundry.

Model Family Mixed / Preview

2.1Seven new MAI models

Headlined by MAI-Thinking-1 — Microsoft AI's first reasoning model: 35B active parameters, 256K context, trained from scratch on clean, commercially-licensed data. Human raters prefer it to Sonnet 4.6, and it matches Opus 4.6 on SWE Bench Pro coding. Also: MAI-Image-2.5 (+flash, live in PowerPoint and rolling into OneDrive), MAI-Transcribe-1.5 (43 languages), MAI-Voice-2 (+flash, 15+ new languages) and MAI-Code-1 (in Copilot + VS Code). MAI models will also be available on Fireworks AI, Baseten and OpenRouter.

MAI models announcement
Customization Private Preview

2.2Frontier Tuning

Applies reinforcement learning inside the customer's compliance boundary so agents learn how the business actually works — using their own data, domain knowledge and workflows in a loop that keeps sharpening as agents run.

Frontier Tuning blog
Model Catalog GA

2.3Fireworks AI on Foundry

Fireworks AI is now generally available on Microsoft Foundry, giving developers a single platform experience — with enterprise governance and Azure data residency — regardless of which model they choose.

Foundry models announcement

Agents

Agent Runtime Preview

Instant-on sandboxes per session with isolated execution, persistent memory and elastic scale — positioned as "the primitive for agents the way containers were for cloud-native apps."

Foundry agents announcement

An always-on personal work agent built on OpenClaw + Work IQ, rolling out first to Frontier customers. It lives in the tools people already use — Teams and Outlook — and proactively handles meeting prep, scheduling conflicts and routine tasks without being asked.

Microsoft Scout blog

Developer Tools & Devices

3 ANNOUNCEMENTS

Agent-native development from the desktop down to the silicon — and a Windows that's becoming an agent-native runtime.

Desktop App Preview

4.1GitHub Copilot app

An agent-native desktop experience. Start from an idea, issue or PR, orchestrate multiple agent sessions in parallel (each on its own git worktree), and keep changes moving through review, CI and merge — Copilot executes while the developer stays in control.

GitHub blog
Hardware Later 2026 · US

4.2Surface RTX Spark Dev Box

Built for sustained workloads — long-running training jobs, agentic pipelines and local fine-tuning. Powered by NVIDIA RTX Spark: up to 1 petaflop of AI compute and 128 GB unified memory, running up to 120B-parameter LLMs with up to 1M-token context locally. WSL2 with native GPU passthrough and full CUDA pre-configured; VS Code and Copilot pre-installed.

Windows devices blog
OS Runtime Preview

4.3Microsoft Execution Containers (MXC)

OS-enforced sandboxed environments for agents on Windows — describe requirements once and Windows enforces them everywhere agents run. Already powering OpenClaw on Windows, and NVIDIA's OpenShell secure runtime builds on MXC with policy management, inference routing and PII obfuscation.

Windows at Build 2026

Security & Governance

3 ANNOUNCEMENTS

A control plane, an open trust stack, and an agentic security system — governance designed to keep pace with agent adoption.

Control Plane Available

6.1Agent 365 for local agents

Extends Entra, Defender and Purview into a single control plane to observe, govern and secure agents across the estate — regardless of where they're hosted or what framework they're built on.

Agent 365 announcement
Open Trust Stack Open Source

6.2ASSERT + Agent Control Specification

An open, end-to-end trust stack for AI agents on any framework, anchored by two open-source projects: ASSERT (Adaptive Spec-driven Scoring for Evaluation and Regression Testing) for policy-driven safety evaluation, and the Agent Control Specification to standardize where and how controls apply in the agent loop.

ASSERT Agent Control Spec
Agentic Security New

6.3Codename MDASH

A multi-model agentic security system that deploys 100+ agents to find exploitable bugs by reasoning about data flow, business logic and exploit chains — delivering context-aware fixes directly in the Defender Portal.

MDASH announcement
